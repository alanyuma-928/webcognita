## 🎨 Architectural Specifications & Design System

This site is engineered as a lightweight, zero-dependency, single-file deployment optimized for rapid delivery via **GitHub Pages**.

### Color Palette (EXW Blue-on-Creme)
Designed for high visibility across diverse lighting environments (outdoor/mobile-first) while maintaining strict WCAG 2.1 AA contrast compliance (>13:1 ratio for primary text).

| Variable | Token | Color Hex | Contrast Ratio / Purpose |
| :--- | :--- | :--- | :--- |
| `--bg-primary` | Soft Creme | `#FAF7F2` | Primary Canvas |
| `--bg-card` | Crisp White | `#FFFFFF` | Card Containers |
| `--bg-box` | Light Blue Tint | `#E8EEF5` | Accent Callouts |
| `--text-main` | Deep Navy | `#0B1D3A` | Main Body Text (15.2:1 Ratio) |
| `--text-sub` | Navy Blue | `#1A365D` | Secondary Headers (11.8:1 Ratio) |

## 🚀 Local Development & Deployment

### Quick Start
Because this repository utilizes a zero-build architecture, local testing requires no package managers or build tools:

1. Clone the repository:
   ```bash
   git clone https://github.com/alanpruitt/webcognita.git
   cd webcognita
   ```
2. Open `index.html` in any modern web browser or start a lightweight local server:
   ```bash
   python3 -m http.server 8000
   ```

### GitHub Pages Deployment
1. Push changes to the `main` branch.
2. Ensure GitHub Pages is configured under **Settings > Pages** to deploy from `Branch: main` / `/ (root)`.
